Infrastructure réseau & système sécurisée

Contexte : PPE AFORP (octobre 2025) : concevoir et déployer un LAN d'entreprise complet avec services Windows et Linux, sécurité périmétrique et accès Internet contrôlé.

Objectifs

  • Mettre en place AD DS, DNS, DHCP sur Windows Server pour le domaine promethee.local.
  • Installer un proxy Squid sur Debian 13 pour filtrage et cache web.
  • Sécuriser les flux via firewall/ACL, NAT et GPO appliquées aux postes clients.

Réalisation

  • Brassage et adressage du réseau 172.16.3.0/24, passerelle 172.16.3.254.
  • Windows Server 172.16.3.250 : rôles AD DS, DNS, DHCP configurés et tests d'intégration poste client.
  • Debian 13 (172.16.3.251) : installation Squid, règles de filtrage, cache et supervision basique des journaux.
  • Firewall : ACL internes/externes, NAT, durcissement de l'admin ; GPO déployées pour proxy et sécurité poste.

Tests

  • Vérification connectivité, obtention IP via DHCP, résolution DNS interne.
  • Connexion domaine avec compte test, application des GPO.
  • Navigation web via proxy Squid et présence des traces dans les logs.

Résultats

Infrastructure fonctionnelle et sécurisée : services AD/DNS/DHCP opérationnels, filtrage web actif, postes intégrés au domaine avec politiques centralisées.

Stack

Windows Server (AD DS, DNS, DHCP), Debian 13 (Squid), firewall/ACL, GPO Windows 10.
